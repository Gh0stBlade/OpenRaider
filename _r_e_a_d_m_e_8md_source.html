1 # OpenRaider
2 
3 [OpenRaider](http://openraider.sourceforge.net) is an Open Source implementation of the classic Tomb Raider Game Engine. It was abandoned in 2003.
4 
5 This project aims to get OpenRaider in a more usable state, maybe one day even being able to play the old Tomb Raider games flawlessly...
6 
7 Currently, my focus is on building and running under Mac OS X.
8 However, my changes should not break the Linux "buildability" :)
9 
10 If you just want to see OpenRaider doing something on your Mac, without installing any of the dependencies needed to build OpenRaider, install [XQuartz](http://xquartz.macosforge.org/trac) (if you don't have it already) and grab the Mac App Bundle from the [most recent release](https://github.com/xythobuz/OpenRaider/releases).
11 If you don't install XQuartz, Mac OS X should prompt you to do so on the first launch of OpenRaider.
12 
13 It seems as if OpenRaider will currently only work on Little-Endian platforms. This is not nice and will hopefully be fixed in the future!
14 
15 ## Configuration
16 
17 OpenRaider needs some configuration files, and level data and assets from custom levels or the Tomb Raider games.
18 These are stored in `~/.OpenRaider`. Running the included `setup.sh` will create/copy the necessary files and directories. The Mac App Bundle will do this for you automatically.
19 
20 You still need to add level files in `~/.OpenRaider/paks/` and add them to `~/.OpenRaider/OpenRaider.init`.
21 Dust off your old Tomb Raider CDs or grab some [custom levels](http://www.aspidetr.com/levels/yvel-woods-v1-5/) from the interwebs.

30 ## Dependencies
31 
32 On Mac OS X 10.9 with [XCode](https://developer.apple.com/xcode/) and [MacPorts](http://www.macports.org) installed, the following should be enough to get all dependencies that are available as port:
33 
34  sudo port install zlib cmake libsdl libsdl_ttf
35 
36 A similar command for the package manager of your favorite Linux Distribution should do the trick.
37 
38 cmake is only needed to build [freealut](https://github.com/vancegroup/freealut), which you'll need to enable sound output on Mac OS X.
39 Get, compile and install freealut like this:
40 
41  git clone git@github.com:vancegroup/freealut.git
42  mkdir build
43  cd build
44  cmake .. -DCMAKE_INSTALL_PREFIX:STRING="/usr/local" -DCMAKE_C_FLAGS:STRING="-O2"
45  make
46  sudo make install
47 
48 Linux Distributions will probably provide an ALUT library ready to install with their package manager, so you won't need to compile freealut.
49 
50 You'll also need the [XQuartz](http://xquartz.macosforge.org/trac) X11 Server if you're on Mac OS X, as Apple is no longer bundling their own X.

52 ## Building
53 
54 If you installed the dependencies using MacPorts, you'll need to have `/opt/local/bin` in your `$PATH` before you can execute make.
55 If you're using Bash, the MacPorts installer should have automatically edited your configuration.
56 
57 Just run `make debug` and run `bin/debug/OpenRaider` for a debug build.
58 `make release` builds a release binary.
59 `make bundle` creates a Mac App Bundle that also runs the setup script, if necessary.

61 ## Usage
62 
63 ### Configuration file
64 
65 OpenRaider will try to load `~/.OpenRaider/OpenRaider.init` or, if it doesn't exist, `OpenRaider.init` from the current directory.
66 Running `setup.sh` will create a minimal configuration in your home directory.
67 
68 The configuration file format is very simple:
69 * Anything following a `#` up to the next `\n` will be ignored.
70 * Lines starting with an `@` mark a "preprocessor" command. The only one currently implemented is `@include`, allowing you to include another file as configuration file.
71 * Lines surrounded with `[]`, eg. `[Engine.OpenGL.Driver]` set the mode for following commands.
72 * Everything else is interpreted as command for the current mode.
73 
74 ### Level PAKs
75 
76 You can use paks from any Tomb Raider version supported and most user made paks for Tomb Raider.
77 Only PHD, TR1, TR2, TR3, and TR4 paks are supported as of this writing.
78 
79 Edit 'Map' lines in your OpenRaider.init as needed.
80 I suggest removing all your Map=... lines to begin with, since no maps are bundled with this release.
81 
82 ### Sound FXs
83 
84 Setting up external sound SFX, for TR2 and TR3 paks only.
85 
86 TR2 and TR3 paks (both have file extension `.tr2`) don't have sound data contained in the pak itself, but instead
87 they share a common SFX file (often MAIN.SFX) for each pak from a given Tomb Raider version.
88 
89 Previously, OpenRaider tried to load an SFX file with the same name as the level file, with `.sfx` added to the end.
90 You had to create a symlink for each level to the SFX file if you were using original Tomb Raider levels, like this:
91 
92  cd ~/.OpenRaider/paks/tr3/
93  cp /mnt/cdrom/data/MAIN.SFX .
94  for i in *.tr2; do ln -s MAIN.SFX $i.sfx; done
95 
96 Now, OpenRaider just tries to load a `MAIN.SFX` in the same folder as the level file.
97 If you want to change this behaviour, this happens around line 1075 of `src/OpenRaider.cpp`.

197 ### Wireframe mode colors
198 
199 * Red (thick) lines are portal outlines
200 * Red (thin) lines are triangle room polygons
201 * Black lines are quadralateral room polygons
202 * White meshes are entities
203 * Blue meshes are sprites
204 * Yellow meshes are room models
205 * Green lines with red endpoints are room bboxes
206 * Pink dots are visibility checks
