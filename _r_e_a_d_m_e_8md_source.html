1 # OpenRaider
2 
3 This is a fork of the [OpenRaider](http://openraider.sourceforge.net) Open Source implementation of the classic Tomb Raider Game Engine. The original project was abandoned in 2003.
4 
5 This fork aims to get OpenRaider in a more usable state, maybe one day even being able to play the old Tomb Raider games flawlessly...
6 
7 If you just want to see OpenRaider doing something on your Mac, without installing any of the dependencies needed to build OpenRaider, grab the Mac App Bundle from the [most recent release](https://github.com/xythobuz/OpenRaider/releases).
8 
9 It seems as if OpenRaider will currently only work on Little-Endian platforms. This is not nice and will hopefully be fixed in the future!

13 ## Configuration
14 
15 OpenRaider needs some configuration files, and level data and assets from custom levels or the Tomb Raider games.
16 These are stored in `~/.OpenRaider`. Running `make setup` will create/copy the necessary files and directories.
17 
18 You still need to add level files in `~/.OpenRaider/paks/`.
19 Dust off your old Tomb Raider CDs or grab some [custom levels](http://www.aspidetr.com/levels/yvel-woods-v1-5/) from the interwebs.

27 ## Dependencies
28 
29 Basically, OpenRaider depends on the following:
30 
31 * OpenGL
32 * SDL2
33 * SDL2-TTF (optional, needed if you want to use TTF fonts)
34 * OpenAL & ALUT
35 * zlib
36 * cmake as build system
37 * libpng (optional)
38 * C++11 compiler
39 
40 On Mac OS X 10.9 with [XCode](https://developer.apple.com/xcode/) and [MacPorts](http://www.macports.org) installed, the following should be enough to get all dependencies that are available as port:
41 
42  sudo port install cmake zlib libsdl2 libsdl2_ttf
43 
44 A similar command for the package manager of your favorite Linux Distribution should do the trick.
45 
46 cmake is also needed to build [freealut](https://github.com/vancegroup/freealut), which you'll need to enable sound output on Mac OS X.
47 Get, compile and install freealut like this:
48 
49  git clone git@github.com:vancegroup/freealut.git
50  mkdir build
51  cd build
52  cmake .. -DCMAKE_INSTALL_PREFIX:STRING="/usr/local" -DCMAKE_C_FLAGS:STRING="-O2"
53  make
54  sudo make install
55 
56 Linux Distributions will probably provide an ALUT library ready to install with their package manager, so you won't need to compile freealut.
57 
58 Sound support is optional now, however. If cmake can’t find OpenAL and ALUT, Audio will be deactivated. If you want to explicitly deactivate it, call cmake with `-DENABLE_AUDIO=NO`.
59 
60 ## Building
61 
62 Do a standard cmake out-of-source build to get a debug configuration and then run the resulting executable:
63 
64  mkdir build && cd build
65  cmake ..
66  make run
67 
68 On Mac OS X, running `make install` will produce a (more or less) distributable OpenRaider.app bundle.
69 
70 You can run a static analysis with cppcheck using `make check`, `make checkFull` or `make checkConfig`.
71 
72 A Doxygen API documentation can be created with `make doc`.
73 
74 `make clean` should remove all intermediary files. `make run` will run the binary.
75 
76 You can also generate working XCode Project files with `cmake -G Xcode ..`.

78 ## Usage
79 
80 ### Configuration file
81 
82 OpenRaider will try to load `~/.OpenRaider/OpenRaider.ini` or, if it doesn't exist, `OpenRaider.ini` from the current directory.
83 Running `make setup` will create a minimal configuration in your home directory.
84 
85 The configuration file format is very simple:
86 * Every line will be executed as in-game console command
87 * The # character marks the beginning of a single-line comment
88 
89 ### Level PAKs
90 
91 You can use paks from any classic Tomb Raider Game (1 - 5) and most user made paks for Tomb Raider. Tomb Raider 5 support is however still very limited.
92 
93 ### Sound FXs
94 
95 TR2 and TR3 paks (both have file extension `.tr2`) don't have sound data contained in the pak itself, but instead
96 they share a common SFX file (often MAIN.SFX) for each pak from a given Tomb Raider version.
97 
98 OpenRaider tries to load a `MAIN.SFX` from the same folder as the selected level file.

116 ### Console/Config Commands
117 
118 Every available command should be listed in the in-game help. Just type `help` in the OpenRaider console, which can be activated by default with the backquote key.
119 
120 ### Tomb Raider Level Editor Font support
121 
122 OpenRaider can read Font.pc files used by the TRLE. If the fonts glyph positions match the TR4 defaults, only a Font.pc file is required. If the positions differ, you also need a [Leikkuri/Cutter](http://trep.trlevel.de/en/downloads.html) preset file (`.lps`).
123 
124 TRLE Font support will automatically be built and used when selecting a font in your config file ending with `.pc`.
125 
126 If a `.lps` is required, give it the same name as your font file.
127 
128 I’ve made [a small writeup on my website](http://xythobuz.de/2014_06_14_trle_font.html) about this.
129 
130 ### Wireframe mode colors
131 
132 * Red (thick) lines are portal outlines
133 * Red (thin) lines are triangle room polygons
134 * Black lines are quadralateral room polygons
135 * White meshes are entities
136 * Blue meshes are sprites
137 * Yellow meshes are room models
138 * Green lines with red endpoints are room bboxes
139 * Pink dots are visibility checks
