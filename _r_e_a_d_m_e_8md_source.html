1 # OpenRaider
2 
3 [OpenRaider](http://openraider.sourceforge.net) is an Open Source implementation of the classic Tomb Raider Game Engine. It was abandoned in 2003.
4 
5 This project aims to get OpenRaider in a more usable state, maybe one day even being able to play the old Tomb Raider games flawlessly...
6 
7 If you just want to see OpenRaider doing something on your Mac, without installing any of the dependencies needed to build OpenRaider, grab the Mac App Bundle from the [most recent release](https://github.com/xythobuz/OpenRaider/releases).
8 
9 It seems as if OpenRaider will currently only work on Little-Endian platforms. This is not nice and will hopefully be fixed in the future!
10 
11 ## Configuration
12 
13 OpenRaider needs some configuration files, and level data and assets from custom levels or the Tomb Raider games.
14 These are stored in `~/.OpenRaider`. Running the included `setup.sh` will create/copy the necessary files and directories.
15 
16 You still need to add level files in `~/.OpenRaider/paks/` and add them to `~/.OpenRaider/OpenRaider.init`.
17 Dust off your old Tomb Raider CDs or grab some [custom levels](http://www.aspidetr.com/levels/yvel-woods-v1-5/) from the interwebs.

26 ## Dependencies
27 
28 Basically, OpenRaider depends on the following:
29 
30 * OpenGL
31 * SDL2 & SDL2-TTF
32 * OpenAL & ALUT
33 * Posix Threads
34 * zlib
35 * cmake as build system
36 
37 On Mac OS X 10.9 with [XCode](https://developer.apple.com/xcode/) and [MacPorts](http://www.macports.org) installed, the following should be enough to get all dependencies that are available as port:
38 
39  sudo port install cmake zlib libsdl2 libsdl2_ttf
40 
41 A similar command for the package manager of your favorite Linux Distribution should do the trick.
42 
43 cmake is also needed to build [freealut](https://github.com/vancegroup/freealut), which you'll need to enable sound output on Mac OS X.
44 Get, compile and install freealut like this:
45 
46  git clone git@github.com:vancegroup/freealut.git
47  mkdir build
48  cd build
49  cmake .. -DCMAKE_INSTALL_PREFIX:STRING="/usr/local" -DCMAKE_C_FLAGS:STRING="-O2"
50  make
51  sudo make install
52 
53 Linux Distributions will probably provide an ALUT library ready to install with their package manager, so you won't need to compile freealut.

55 ## Building
56 
57 Do a standard cmake out-of-source build and then run the resulting executable:
58 
59  mkdir build
60  cd build
61  cmake -DCMAKE_BUILD_TYPE=Debug ..
62  make
63  ./src/OpenRaider
64 
65 On Mac OS X, running `make install` will produce a (more or less) distributable OpenRaider.app bundle.
66 
67 To build all Unit Tests run `make tests`, to build and run most of them use `make testRun`.
68 
69 To run a TombRaider level loader regression test use `make testReg`. The results will be put into log files in your build folder.
70 
71 You can run a static analysis with cppcheck using `make check`, `make checkFull` or `make checkConfig`.
72 
73 A Doxygen API documentation can be created with `make doc`.
74 
75 `make clean` should remove all intermediary files. `make run` will run the binary.

77 ## Usage
78 
79 ### Configuration file
80 
81 OpenRaider will try to load `~/.OpenRaider/OpenRaider.init` or, if it doesn't exist, `OpenRaider.init` from the current directory.
82 Running `make setup` or the `cmake/setup.sh` script will create a minimal configuration in your home directory.
83 
84 The configuration file format is very simple:
85 * Anything following a `#` up to the next `\n` will be ignored.
86 * Lines starting with an `@` mark a "preprocessor" command. The only one currently implemented is `@include`, allowing you to include another file as configuration file.
87 * Lines surrounded with `[]`, eg. `[Engine.OpenGL.Driver]` set the mode for following commands.
88 * Everything else is interpreted as command for the current mode.
89 
90 ### Level PAKs
91 
92 You can use paks from any Tomb Raider version supported and most user made paks for Tomb Raider.
93 Only PHD, TR1, TR2, TR3, and TR4 paks are supported as of this writing.
94 
95 ### Sound FXs
96 
97 Setting up external sound SFX, for TR2 and TR3 paks only.
98 
99 TR2 and TR3 paks (both have file extension `.tr2`) don't have sound data contained in the pak itself, but instead
100 they share a common SFX file (often MAIN.SFX) for each pak from a given Tomb Raider version.
101 
102 Previously, OpenRaider tried to load an SFX file with the same name as the level file, with `.sfx` added to the end.
103 You had to create a symlink for each level to the SFX file if you were using original Tomb Raider levels, like this:
104 
105  cd ~/.OpenRaider/paks/tr3/
106  cp /mnt/cdrom/data/MAIN.SFX .
107  for i in *.tr2; do ln -s MAIN.SFX $i.sfx; done
108 
109 Now, OpenRaider just tries to load a `MAIN.SFX` in the same folder as the level file.
110 If you want to change this behaviour, this happens around line 1075 of `src/OpenRaider.cpp`.

209 ### Wireframe mode colors
210 
211 * Red (thick) lines are portal outlines
212 * Red (thin) lines are triangle room polygons
213 * Black lines are quadralateral room polygons
214 * White meshes are entities
215 * Blue meshes are sprites
216 * Yellow meshes are room models
217 * Green lines with red endpoints are room bboxes
218 * Pink dots are visibility checks
